# 常用踩坑记录

## WSL 与 Windows 路径问题

### 问题
WSL 和 Windows 路径格式不兼容，导致文件找不到。

### 解决
- WSL 路径: `/home/jinlong/...`
- Windows 路径: `C:\Users\...`
- 不要混用，使用 WSL 路径格式

### 场景
在 WSL 中运行 Windows 程序（如 VS Code）可能路径出问题。

## npm install 失败

### 问题
安装依赖时报错，可能是网络问题或权限问题。

### 解决
```bash
# 使用代理
npm install --proxy http://127.0.0.1:9999

# 清理缓存重试
npm cache clean --force
rm -rf node_modules
npm install
```

## Git 提交冲突

### 问题
多设备同步时出现冲突。

### 解决
```bash
# 查看冲突
git status

# 保留本地版本
git checkout --ours 文件名
# 或保留远程版本
git checkout --theirs 文件名

# 标记解决
git add 文件名
git commit -m "resolve conflict"
```

## Claude Code 权限问题

### 问题
某些命令需要权限才能执行。

### 解决
- 使用 `allow` 规则添加权限到 `~/.claude/settings.json`
- 示例: `"Bash(git push:*)"`

## Chrome DevTools Protocol 连接失败

### 问题
test_chrome.js 等脚本无法连接 Chrome。

### 解决
1. 确保 Chrome 以 `--remote-debugging-port=9222` 启动
2. 检查防火墙设置
3. 确认端口未被占用

## 代理导致的问题

### 问题
启用代理后，某些国内服务无法访问。

### 解决
```bash
# 使用 PROXY_CN=direct 绕过代理
PROXY_CN=direct npm install
```

或在 Claude Code settings 中配置 `PROXY_CN: direct`

## Obsidian Git 插件 SSH 认证失败

### 问题
Obsidian 报错 `Host key verification failed. Could not read from remote repository`

### 原因
- Obsidian 运行在 Windows 环境，用的是 Windows Git，不是 WSL Git
- WSL 有 SSH key，但 Windows Git 没有
- Remote URL 用 SSH 方式 (`git@github.com:...`)

### 解决
修改 `.git/config`，把 SSH 方式改成 HTTPS：

```bash
# 改 remote URL
git remote set-url origin https://github.com/用户名/仓库名.git
```

或直接编辑 `.git/config`，把：
```
url = git@github.com:用户名/仓库名.git
```
改成：
```
url = https://github.com/用户名/仓库名.git
```

### 教训
Obsidian + Git 插件在 Windows 下工作，WSL SSH key 无效，统一用 HTTPS 方式。

## WSL2 与 Windows Obsidian Vault 不同步

### 问题
Obsidian vault 在 WSL 是 `/home/jinlong/obsidian-vault`，在 Windows 是 `C:\Users\JINLONG\obsidian-vault`，两个不同步。

### 原因
- Obsidian 是 Windows 应用，天然在 Windows 运行
- WSL 和 Windows 是两个文件系统

### 解决
- **推荐**：vault 统一放 Windows，通过 GitHub 同步
- **不要**在 WSL 里操作 vault
- WSL 访问 Windows 文件用 `/mnt/c/...` 路径

### 推荐架构
- Obsidian vault: `C:\Users\JINLONG\obsidian-vault`
- Claude Code 项目: WSL 里工作
- 两台电脑通过 GitHub 同步 vault
