## WSL 与 Windows 路径问题

### 问题
WSL 和 Windows 路径格式不兼容，导致文件找不到。

### 解决
- WSL 路径: `/home/jinlong/...`
- Windows 路径: `C:\Users\...`
- 不要混用，使用 WSL 路径格式

### 场景
在 WSL 中运行 Windows 程序（如 VS Code）可能路径出问题。

## npm install 失败

### 问题
安装依赖时报错，可能是网络问题或权限问题。

### 解决
```bash
# 使用代理
npm install --proxy http://127.0.0.1:9999

# 清理缓存重试
npm cache clean --force
rm -rf node_modules
npm install
```

## Obsidian Git 插件 SSH 认证失败

### 问题
Obsidian 报错 `Host key verification failed. Could not read from remote repository`

### 原因
- Obsidian 运行在 Windows 环境，用的是 Windows Git，不是 WSL Git
- WSL 有 SSH key，但 Windows Git 没有
- Remote URL 用 SSH 方式 (`git@github.com:...`)

### 解决
修改 `.git/config`，把 SSH 方式改成 HTTPS：

```bash
# 改 remote URL
git remote set-url origin https://github.com/用户名/仓库名.git
```

或直接编辑 `.git/config`，把：
```
url = git@github.com:用户名/仓库名.git
```
改成：
```
url = https://github.com/用户名/仓库名.git
```

### 教训
Obsidian + Git 插件在 Windows 下工作，WSL SSH key 无效，统一用 HTTPS 方式。

## WSL2 与 Windows Obsidian Vault 不同步

### 问题
Obsidian vault 在 WSL 是 `/home/jinlong/obsidian-vault`，在 Windows 是 `C:\Users\JINLONG\obsidian-vault`，两个不同步。

### 原因
- Obsidian 是 Windows 应用，天然在 Windows 运行
- WSL 和 Windows 是两个文件系统

### 解决
- **推荐**：vault 统一放 Windows，通过 GitHub 同步
- **不要**在 WSL 里操作 vault
- WSL 访问 Windows 文件用 `/mnt/c/...` 路径

### 推荐架构
- Obsidian vault: `C:\Users\JINLONG\obsidian-vault`
- Claude Code 项目: WSL 里工作
- 两台电脑通过 GitHub 同步 vault

## HTTPS 方式 Git Push 需要认证

### 问题
使用 HTTPS 方式 push 时，报错：
```
fatal: could not read Username for 'https://github.com': No such device or address
```

### 原因
HTTPS 方式需要 GitHub 认证，但没有配置凭证。

### 解决
**方法 1: 使用 Obsidian Git 插件推送**
- 在 Obsidian 侧边栏点击 Git 图标
- 点击 Commit 和 Push 按钮

**方法 2: 使用 GitHub Personal Access Token**
1. 创建 Token：https://github.com/settings/tokens
2. Generate new token (classic)
3. 勾选 `repo` 权限
4. push 时用 token 当作密码

**方法 3: 继续用 SSH 方式**
- 如果 SSH key 已配置好，可以改回 SSH 方式
- WSL 环境用 SSH 更方便

### 推荐
- WSL 环境：用 SSH 方式（`git@github.com:...`）
- Windows Obsidian 插件：用 HTTPS 方式

## Windows PowerShell Git 代理配置

### 问题
Git 在 PowerShell 中无法连接 GitHub，报错：
```
Failed to connect to github.com port 443
```

### 解决
```powershell
git config --global http.proxy http://127.0.0.1:9999
git config --global https.proxy http://127.0.0.1:9999
```

### 验证
```powershell
git config --global --list
```

## Windows PowerShell curl 参数问题

### 问题
PowerShell 的 `curl` 是 `Invoke-WebRequest` 的别名，不支持 `-L` 参数：
```
找不到与参数名称"L"匹配的参数
```

### 解决
```powershell
# 使用完整参数名
curl -Proxy "http://127.0.0.1:9999" -o filename "url"

# 或者直接用 Invoke-WebRequest
Invoke-WebRequest -Proxy "http://127.0.0.1:9999" -Uri "url" -OutFile "filename"
```

### 建议
代理环境下直接用 `-Proxy` 参数指定代理服务器。

## Obsidian Git 插件手动安装

### 问题
社区插件市场搜索不到插件。

### 解决
手动下载插件文件到 vault 的 `.obsidian\plugins\obsidian-git\` 目录：

1. 创建目录：
   ```powershell
   mkdir "C:\Users\JINLONG\obsidian-vault\.obsidian\plugins\obsidian-git"
   ```

2. 下载文件（使用代理）：
   ```powershell
   cd "C:\Users\JINLONG\obsidian-vault\.obsidian\plugins\obsidian-git"
   curl -Proxy "http://127.0.0.1:9999" -o manifest.json "https://github.com/denolehov/obsidian-git/releases/download/2.5.0/manifest.json"
   curl -Proxy "http://127.0.0.1:9999" -o main.js "https://github.com/denolehov/obsidian-git/releases/download/2.5.0/main.js"
   curl -Proxy "http://127.0.0.1:9999" -o styles.css "https://github.com/denolehov/obsidian-git/releases/download/2.5.0/styles.css"
   ```

3. 重启 Obsidian，启用插件

## Obsidian Git 插件配置检查清单

### 必须确保的配置
- `autoPullOnBoot`: `true`（启动时自动拉取）
- `differentIntervalCommitAndPush`: `false`（立即推送）

### 检查方法

cat "C:\Users\JINLONG\obsidian-vault\.obsidian\plugins\obsidian-git\data.json"

将完整的 data.json 内容替换，确保所有关键配置正确。

## 重要规则总结

1. **Obsidian Vault 统一用 HTTPS** - 避免 SSH key 问题
2. **WSL 项目用 SSH** - CC_TEST、claude-config

5. **thirdIntervalCommitAndPush 必须为 false** - 否则推送延迟
