
## 工作习惯

1. **代码优先**: 先写代码，再优化
2. **小步快跑**: 每次改动的代码量要小
3. **测试驱动**: 重要功能要先写测试
4. **简洁风格**: 代码要简洁，不必要的复杂度不要有

## 常用工具

- **终端**: bash
- **编辑器**: VS Code
- **AI 助手**: Claude Code
- **版本控制**: Git

## 编码风格

- 变量命名用英文驼峰或下划线
- 函数名用动词开头
- 类名用名词，首字母大写
- 常量全大写，下划线分隔

## 常见错误处理

- 网络错误：检查代理配置
- 路径错误：检查 WSL vs Windows 路径格式
- 权限错误：检查文件权限和 sudo
